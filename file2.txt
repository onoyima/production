üîÑ WORKFLOW 
1.	Student logs in ‚Üí Applies for Exeat (Medical) ‚Üí CMD sees it ‚Üí Approves
IF Non-Medical : Student logs in ‚Üí Applies for Exeat
2.	Request sent to Deputy Dean ‚Üí Sends consent link to parent
3.	Parent approves via WhatsApp/Email/SMS ‚Üí Approves(Phone call ->Deputy Dean Approves)
4.	Dean approves ‚Üí Hostel Admin signs out ‚Üí Security signs out
5.	Student returns ‚Üí Security & Hostel Admin sign in ‚Üí Closed.

Detailed User Journey Diagrams
1.	Student Journey
o	Login ‚Üí Apply for Exeat (Medical/Non-Medical) ‚Üí If Medical, CMD Approves ‚Üí Request Sent to Deputy Dean ‚Üí Parent Consent ‚Üí Dean Approval ‚Üí Hostel Admin Sign Out ‚Üí Security Sign Out ‚Üí Student Leaves ‚Üí Return ‚Üí Security Sign In ‚Üí Hostel Admin Sign In ‚Üí Mark Completed

4.	Dean Journey
o	Login ‚Üí View Parent-Approved Requests ‚Üí Approve/Decline ‚Üí Notify Hostel Admin
5.	Hostel Admin / Security Journey
o	Login ‚Üí Receive Notification of Approved Exeat ‚Üí Sign Out Student ‚Üí Await Return ‚Üí Sign In Student ‚Üí Close Request
